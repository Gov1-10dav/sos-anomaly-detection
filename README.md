# Алгоритм поиска аномальных респондентов

## Описание алгоритма
Данный алгоритм выявляет респондентов с аномально высокой поисковой активностью по отдельным брендам в рамках одного дня. Аномалией считается экстремальный всплеск активности, который сильно выделяется на фоне типичного поведения пользователей.

## Логика расчета
1. **Агрегация OTS:** Для каждой уникальной комбинации респондента, дня, категории и бренда рассчитывается дневной OTS. Если в данных присутствует столбец `Weight`, количество запросов умножается на дневной вес. Если столбца нет, OTS равен количеству строк. Учитываются только строки, входящие в поставку (`BrandinDelivery = 1`).
2. **Глобальный порог (Защита от малых выборок):** Рассчитывается 95-й перцентиль OTS по всему датасету. Это значение отсекает обычных пользователей с естественной, но не аномальной активностью, защищая алгоритм от случайных срабатываний на редких брендах с малым общим числом запросов.
3. **Локальный порог (Брендовый):** Для каждого отдельного бренда вычисляется свой 99-й перцентиль OTS. Это позволяет найти экстремальные выбросы активности именно внутри конкретного бренда, учитывая его специфику популярности.

## Правило удаления
Респондент признается аномальным в конкретный день, если его `daily_ots` по какому-либо бренду одновременно строго превышает:
* Локальный порог бренда (99-й перцентиль)
* Глобальный порог (95-й перцентиль)

Если найдено превышение хотя бы по одному бренду, респондент удаляется целиком за эти исследовательские сутки (все его запросы в этот день игнорируются при дальнейшем анализе). 
Данный подход не использует хардкод и автоматически подстраивается под любые новые данные.
